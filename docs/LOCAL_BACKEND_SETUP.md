# Local MySQL Backend Setup

The dashboard now reads from the MySQL tables generated by `main.py`:

- `bot_detection_results`
- `bot_rule_hits`

SQLite and CSV import are no longer part of the active backend path.

## 1. Generate MySQL Data

From the project root:

Edit `.env`:

```env
ANTIBOT_DB_PASSWORD=your_password
ANTIBOT_DB_NAME=anti_bot
ANTIBOT_SOURCE_TABLE=fingerprints_sample
ANTIBOT_SOURCE_JSON_FIELD=rest_json
ANTIBOT_BATCH_SIZE=10000
```

Then run:

```powershell
.\.venv\Scripts\python.exe main.py
```

This writes analysis results into MySQL.

## 2. Start Backend

From `backend`, edit `.env`:

```env
ANTIBOT_DB_PASSWORD=your_password
ANTIBOT_DB_NAME=anti_bot
```

Then run:

```powershell
npm install
npm run dev
```

Optional backend variables:

```powershell
$env:PORT="8080"
$env:ANTIBOT_RESULT_TABLE="bot_detection_results"
$env:ANTIBOT_RULE_HIT_TABLE="bot_rule_hits"
$env:CORS_ORIGIN="*"
```

You can also use a full connection URL:

```powershell
$env:ANTIBOT_DB_URL="mysql://root:your_password@localhost:3306/anti_bot"
```

## 3. Start Frontend

From `frontend`:

```powershell
npm install
npm run dev
```

The Vite proxy forwards `/api` to `VITE_BACKEND_ORIGIN`.

Default `frontend/.env.local`:

```env
VITE_BACKEND_ORIGIN=http://127.0.0.1:8080
VITE_API_TIMEOUT_MS=6000
VITE_USE_MOCK=false
VITE_API_FALLBACK_TO_MOCK=true
```

## 4. Main API Endpoints

- `GET /health`
- `GET /api/v1/stats/defense`
- `GET /api/v1/stats/purity`
- `GET /api/v1/stats/spoofing`
- `GET /api/v1/stats/locations`
- `GET /api/v1/logs/live`
- `GET /api/v1/logs/scatter`
- `GET /api/v1/detections`
- `GET /api/v1/detections/:id`
- `GET /api/v1/rules/top`
- `POST /api/v1/agent/query`

The frontend uses these APIs directly; the old SQLite tables and `fingerprint_logs` path are retired.
